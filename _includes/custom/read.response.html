Return a single {{include.resource}} for the specified id{{include.content}}.

All requests SHALL contain a valid ‘Authorization’ header and SHALL contain an ‘Accept’ header with at least one of the following application/json+fhir or application/xml+fhir.

Http Headers

Accept: application/json+fhir
Authorization: <as mandated by local standard>

1.1. Response

If no {{include.resource}} resource is found an OperationOutcome will be returned.

Response class:

  • (Status 200): successful operation
  • (Status 404): resource not found
  • (Status 410): resource deleted
